Explanation (short)

RSA: we generate small primes p,q, compute n=p*q, choose public exponent e and compute private exponent d. We convert each character to its numeric code (0–255) and encrypt each char as c = m^e mod n.

Reference distribution attack: To decode without d, for each ciphertext block c we brute-force candidate plaintext bytes m (printable ASCII 32–126) and keep those where m^e mod n == c. If multiple candidates match, we choose the most probable character using an English frequency table. This demonstrates why per-character RSA with small modulus is vulnerable to statistical attacks.
